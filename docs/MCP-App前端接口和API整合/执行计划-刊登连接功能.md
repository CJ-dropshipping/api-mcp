## 一、背景与目标

用户希望在 MCP 对话中，让 AI 帮助完成"刊登商品到店铺"和"连接（关联）商品"两个操作，具体方式是：

1. AI 生成带有特定 URL 参数的商品详情页链接
2. 用户在浏览器中打开该链接后，页面自动触发对应弹窗/操作流程

目标效果：
- `open_listing_page` → 生成商品详情页 URL，带 `?list=1`，打开后自动弹出刊登 iframe 弹窗
- `open_product_connect_page` → 生成商品详情页 URL，带 `?connect=1`，打开后自动触发连接（关联店铺）流程

---

## 二、现状分析

### 2.1 MCP 工具现状（CJMCPAPP）

| 工具名 | 当前生成 URL | 问题 |
|--------|------------|------|
| `open_listing_page` | `{webBase}/myCJ.html#/listing?pid=${productId}` | 跳转到"我的账号-刊登"列表页，不是商品详情页刊登弹窗 |
| `open_product_connect_page` | `{webBase}/myCJ.html#/productConnect` | 跳转到通用商品关联页，缺少商品上下文（没有 productId/sku） |

### 2.2 前端 URL 参数支持现状（mf-components）

| 功能 | 触发函数 | 现有 URL param 支持 |
|------|---------|-------------------|
| 刊登 | `hookList.openListModal()` | ✅ 已支持 `?list=1`（`use-eidt-list.ts` 的 `useEffect` 自动检测并触发） |
| 连接 | `hookConnect.goConnection()` | ❌ 不支持 URL param，只能靠按钮点击或 next-step 存储触发 |

### 2.3 关键代码路径

**刊登（已支持 `?list=1`）——`use-eidt-list.ts` 第55~67行**：
```typescript
useEffect(() => {
  window.listIframe && window.listIframe.createIframe()
  const list = getQueryVariable('list')
  if (list == 1) {
    if (document.referrer.includes('login.html') && !isLogin) {
      removeListUrlQuery()
      return
    }
    openListModal()
  }
}, [])
```

**连接（当前无 URL param 支持）——`use-connect.ts` 第22~45行**：
```typescript
const goConnection = () => {
  const { sku } = productInfo
  if (!isLogin) {
    addNextStep('connection')
    goLogin()
    return
  }
  APIS.connectionProduct({ accountId: ..., shopStatus: 1 }).then(([, res]) => {
    if (res.code == 200) {
      window.open(`/my.html#/products-connection/pending-connection?sku=${Base64.encode(sku)}`)
    } else {
      window.open('/my.html#/authorize/Shopify')
    }
  })
}
```

## 四、执行步骤（共 4 个文件变更）

### Step 1：修改前端 `use-connect.ts`（mf-components 项目）

**文件路径**：
`packages/mf-mycj-components/src/components/CjProductDetail/components/ProductInfo/Operations/use-connect.ts`

**变更目标**：新增对 `?connect=1` URL 参数的自动检测，检测到后调用 `goConnection()`，并在处理后清理参数。

**新增 import**：
```typescript
import { getQueryVariable } from '@/common'
import { useEffect } from 'react'
```

**新增函数 `removeConnectUrlQuery`**（参考 `removeListUrlQuery` 的实现）：
```typescript
const removeConnectUrlQuery = () => {
  const url = new URL(window.location.href)
  const params = new URLSearchParams(url.search)
  params.delete('connect')
  url.search = params.toString()
  window.history.pushState(null, '', url.toString())
}
```

**新增 `useEffect`**（放在 `return` 语句之前）：
```typescript
useEffect(() => {
  const connect = getQueryVariable('connect')
  if (connect == 1) {
    if (document.referrer.includes('login.html') && !isLogin) {
      removeConnectUrlQuery()
      return
    }
    goConnection()
  }
}, [])
```

**注意**：`goConnection` 函数在调用时需要 `productInfo.sku` 已加载。由于 `useEffect` 在组件 mount 后执行，`productInfo` 可能尚未从接口返回，需参考 `openListModal` 的重试机制——不过 `openListModal` 等待的是 DOM 元素 `#manualListIframe`；`goConnection` 直接调用 API，`productInfo.sku` 依赖 context 加载。

**关于异步数据加载的处理**：`goConnection` 内部使用 `productInfo.sku`，而 `productInfo` 来自 `useProductInfo()` context。建议在 `useEffect` 中对 `isLogin` 和 `productInfo.sku` 都做就绪检测：

```typescript
useEffect(() => {
  const connect = getQueryVariable('connect')
  if (connect == 1) {
    if (document.referrer.includes('login.html') && !isLogin) {
      removeConnectUrlQuery()
      return
    }
    // 数据可能尚未加载完成，等待 sku 就绪后再触发
    if (!productInfo?.sku) {
      // 数据未就绪，等下一次 render（productInfo 变化时 useEffect 会重新运行）
      return
    }
    goConnection()
  }
}, [productInfo?.sku])  // ← 依赖 sku 变化，确保数据就绪后触发
```

**最终 `return` 增加导出**（如有需要向外暴露）：
```typescript
return {
  goConnection,
}
```
（无需新增，保持原有 return 不变）

## 七、注意事项

1. **mf-components 修改属于前端组件库**，修改后需发版（或通过内部分支提测）才能在实际线上环境生效，测试时可在本地 link 调试。

2. **`?connect=1` 的 useEffect 依赖**：设置 `[productInfo?.sku]` 依赖，确保 sku 就绪后才触发，避免 API 调用时 sku 为空。

3. **登录态处理**：与 `?list=1` 一致，若从 `login.html` 返回且未登录，应清理参数并 return，避免死循环。

4. **backward compatibility**：原 `open_listing_page` 的 `shopId` 参数不再使用，可安全移除（原 URL 格式 `/myCJ.html#/listing` 是后台路由，不影响商品详情页，移除无副作用）。

5. **原 `open_product_connect_page` 无 `productId` 参数**，修改后需要 `productId` 为必填，MCP 工具的 `required` 需同步更新。

### 根本原因（已通过日志 + 官方 ext-apps SDK 源码确认）：
- ChatGPT 对 `resourceUri` HTML 只读取一次（第一次 `show_product_list` 调用），后续复用缓存
- 原方案用 `window.__INITIAL_DATA__` 注入数据，但 HTML 不重新读取 → 数据永不更新
- 官方 MCP Apps SDK 的正确机制：HTML 发起 `ui/initialize` 握手 → ChatGPT 对每次工具调用推送 `ui/notifications/tool-result` → iframe 收到后用 `structuredContent` 重新渲染
- 原 HTML 没有实现 MCP Apps 协议握手 → ChatGPT 不推送 `tool-result` → 数据始终是第一次的
### 任务1: resources/index.ts 导出 getProductListCache()
### 任务2: product.tool.ts show_product_list 返回 structuredContent + 固定 URI（不含时间戳）

### 任务4: product-tool.test.ts 添加 2 个新测试（有缓存/无缓存）
### 任务5: 运行测试（96/99 通过，3 个预存 auth-tool 失败不变）
### 任务6: 构建验证

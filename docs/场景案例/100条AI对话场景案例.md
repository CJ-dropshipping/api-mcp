## 认证与登录

### 案例 01 — 首次登录

**中文：**
> 请帮我登录 CJ 账号

**English：**
> Help me log in to my CJ account

**触发工具：** `show_login_form`
**说明：** AI 会弹出 CJ 登录页面，用户在页面中输入账号密码完成登录，Session 自动持久化。

---

### 案例 02 — 检查登录状态

**中文：**
> 我现在是否已经登录了 CJ？

**English：**
> Am I currently logged in to CJ?

**触发工具：** `check_login_status`
**说明：** AI 检查当前 Session Token 的有效性，并返回登录状态和账号信息。

---

### 案例 03 — 使用 API Key 验证凭据

**中文：**
> 帮我用 API Key 验证 CJ 的访问凭据，我的 accessKey 是 xxxxxxxx，secretKey 是 yyyyyyyy

**English：**
> Verify my CJ API credentials — accessKey: xxxxxxxx, secretKey: yyyyyyyy

**触发工具：** `verify_credentials`
**说明：** 适用于 ChatGPT 等远程模式，通过 API Key 换取 Access Token，无需 UI 登录。

---

### 案例 04 — 退出登录

**中文：**
> 帮我退出 CJ 登录

**English：**
> Please log me out of CJ

**触发工具：** `logout`
**说明：** 清除本地 Session Token，下次使用需重新登录。

---

### 案例 05 — 等待登录完成

**中文：**
> 我已经在登录页面填好了，请等我完成登录

**English：**
> I've filled in the login form, please wait for me to finish logging in

**触发工具：** `wait_for_login`
**说明：** AI 进入等待状态，轮询登录结果，登录成功后自动继续后续操作。

---

### 案例 06 — 获取 OAuth 授权链接

**中文：**
> 帮我获取 CJ 的 OAuth 授权链接，我想在第三方平台完成授权

**English：**
> Get the CJ OAuth authorization URL so I can authorize on a third-party platform

**触发工具：** `get_authorize_url`
**说明：** 返回 OAuth 2.0 授权链接，用于第三方系统集成 CJ 账号授权。

---

### 案例 07 — Session 过期后重新登录

**中文：**
> 刚才报了登录过期的错误，帮我重新登录一下

**English：**
> I got a "session expired" error just now, please help me log in again

**触发工具：** `show_login_form`
**说明：** Token 过期后，AI 自动识别 `AuthExpiredError`，引导用户重新完成登录流程。

---

### 案例 08 — 查看账户设置

**中文：**
> 帮我查看一下我的 CJ 账户设置信息

**English：**
> Show me my CJ account settings

**触发工具：** `get_account_settings`
**说明：** 返回账户基本信息，包括用户名、邮箱、账户级别、功能权限等。
